## 🔍 各阶段问题记录

### 第一阶段：项目启动 (Day01-02)

**问题1：FFmpeg 依赖版本冲突**

| 项目 | 详情 |
|------|------|
| **问题描述** | 安装 vcpkg ffmpeg 时出现版本冲突，无法同时安装 dxva2 和 d3d11va 支持 |
| **发现时间** | Day01 |
| **影响范围** | 硬件解码功能 |
| **根本原因** | vcpkg 配置文件中 ffmpeg 的 feature 配置不完整 |
| **解决方式** | 修改 vcpkg.json，明确指定需要的 features |
| **修复时间** | Day02 |

**问题2：Direct3D 11 初始化失败**

| 项目 | 详情 |
|------|------|
| **问题描述** | 在部分 Win10 老版本上 D3D11 设备创建失败 |
| **发现时间** | Day02 |
| **影响范围** | 渲染功能 |
| **根本原因** | D3D_FEATURE_LEVEL 设置过高 |
| **解决方式** | 降低最小特性级别要求，增加特性级别检测 |
| **修复时间** | Day02 |

---

### 第二阶段：开发阶段 (Day03-05)

**问题3：硬件解码初始化失败**

| 项目 | 详情 |
|------|------|
| **问题描述** | DXVA2 硬件解码初始化失败，无法创建硬件设备上下文 |
| **发现时间** | Day03 |
| **影响范围** | 硬件解码功能 |
| **根本原因** | 显卡驱动不支持 DXVA2 或硬件加速被禁用 |
| **解决方式** | 增加软件解码回退机制 |
| **修复时间** | Day04 |

**问题4：音视频不同步**

| 项目 | 详情 |
|------|------|
| **问题描述** | 播放过程中出现音画不同步现象 |
| **发现时间** | Day04 |
| **影响范围** | 播放体验 |
| **根本原因** | PTS 时间戳计算错误，同步阈值设置不合理 |
| **解决方式** | 优化同步算法，调整阈值为 50ms |
| **修复时间** | Day05 |

**问题5：内存泄漏**

| 项目 | 详情 |
|------|------|
| **问题描述** | 长时间播放后内存占用持续增长 |
| **发现时间** | Day05 |
| **影响范围** | 稳定性 |
| **根本原因** | AVFrame 和 AVPacket 未正确释放 |
| **解决方式** | 采用 RAII 模式管理资源，使用智能指针 |
| **修复时间** | Day05 |

---

### 第三阶段：功能开发 (Day06-08)

**问题6：跑马灯文字重叠**

| 项目 | 详情 |
|------|------|
| **问题描述** | 跑马灯文字到达边界时出现重叠显示 |
| **发现时间** | Day06 |
| **影响范围** | 跑马灯功能 |
| **根本原因** | 文字位置计算未考虑文字宽度 |
| **解决方式** | 使用 DirectWrite 测量文字宽度，修正位置计算 |
| **修复时间** | Day06 |

**问题7：多片源切换花屏**

| 项目 | 详情 |
|------|------|
| **问题描述** | 片源切换时出现短暂花屏 |
| **发现时间** | Day07 |
| **影响范围** | 片源切换功能 |
| **根本原因** | 帧队列未清空导致旧帧残留 |
| **解决方式** | 切换前清空帧队列，实现无缝切换 |
| **修复时间** | Day08 |

**问题8：高DPI显示模糊**

| 项目 | 详情 |
|------|------|
| **问题描述** | 在高分辨率显示器上界面元素模糊 |
| **发现时间** | Day08 |
| **影响范围** | UI显示 |
| **根本原因** | 未启用 DPI 感知 |
| **解决方式** | 在 manifest 中添加 DPI 感知声明 |
| **修复时间** | Day08 |

## 🔮 经验教训总结

### 1. 依赖管理
- 库版本必须与头文件严格匹配
- 使用包管理器时注意 feature 配置
- 保留兼容旧版本的方案

### 2. 兼容性设计
- 考虑多平台支持时提前规划
- 硬件解码需要软件回退机制
- DPI 感知需要在早期阶段考虑

### 3. 代码质量
- 资源管理采用 RAII 模式
- 内存泄漏检测应作为常规流程
- 异常处理需要完善

### 4. 构建流程
- 避免硬编码路径
- 构建脚本需要版本控制
- 编译产物位置需要验证
