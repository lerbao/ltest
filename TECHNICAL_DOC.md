## 1. 产品需求文档 (PRD)

### 1.1 产品概述

**产品名称**：VideoPlayer

**产品定位**：基于 mpv 库的高性能 Windows 视频播放器，支持 Windows 7/10/11 多平台运行。

**核心价值**：
- 自动扫描并播放视频目录中的视频文件
- 支持多种视频格式的硬件加速播放
- 跨版本 Windows 兼容，无需重新编译

### 1.2 功能需求

| 需求编号 | 功能描述 | 优先级 | 来源 |
|----------|----------|--------|------|
| FR-001 | 自动扫描 video 目录中的视频文件 | 高 | README.md |
| FR-002 | 支持多种视频格式（MP4, MKV, AVI, MOV 等） | 高 | README.md |
| FR-003 | 硬件解码加速（DXVA2/D3D11VA） | 高 | README.md |
| FR-004 | 音视频同步播放 | 高 | README.md |
| FR-005 | Windows 7/10/11 跨平台兼容 | 高 | README.md |
| FR-006 | 定时自动切换视频（15分钟间隔） | 中 | VideoPlayer.cpp |
| FR-007 | 鼠标点击切换视频 | 中 | VideoPlayer.cpp |
| FR-008 | ESC 键退出程序 | 中 | VideoPlayer.cpp |
| FR-009 | 自动故障恢复（MPV 重置机制） | 中 | VideoPlayer.cpp |

### 1.3 非功能需求

| 需求编号 | 描述 | 指标 |
|----------|------|------|
| NFR-001 | 启动时间 | < 2 秒 |
| NFR-002 | 视频加载时间 | < 1 秒 |
| NFR-003 | 稳定性 | 连续运行 > 24 小时 |
| NFR-004 | 资源占用 | 内存 < 500MB，CPU < 20% |
| NFR-005 | 兼容性 | Windows 7 SP1 / 10 / 11 |

### 1.4 使用场景

| 场景 | 描述 |

## 3. 核心实现原理

### 3.1 跨平台兼容机制

**设计目标**：单一可执行文件支持 Windows 7/10/11

**实现原理**：

```cpp
// 运行时系统版本检测
BOOL IsWindows7() {
    OSVERSIONINFOEXW osvi;
    ZeroMemory(&osvi, sizeof(OSVERSIONINFOEXW));
    osvi.dwOSVersionInfoSize = sizeof(OSVERSIONINFOEXW);
    
    // 使用 VerifyVersionInfo 检测 Windows 7
    // ... 具体实现 ...
}

// 动态 DLL 路径设置
if (IsWindows7()) {
    swprintf(dllPath, ..., L"%s\\..\\lib\\lib_win7", exePath);
} else {
    swprintf(dllPath, ..., L"%s\\..\\lib\\lib_win10", exePath);
}
SetDllDirectoryW(dllPath);
```

**技术要点**：

| 机制 | 说明 |
|------|------|
| **延迟加载** | `/DELAYLOAD:libmpv-2.dll` 延迟 DLL 加载时机 |
| **动态路径** | `SetDllDirectoryW()` 在运行时设置 DLL 搜索路径 |
| **条件检测** | `IsWindows7()` 使用 Windows API 检测系统版本 |
| **并行部署** | 两个 lib 目录同时存在，运行时选择 |

### 3.2 视频播放流程

```
┌─────────────────────────────────────────────────────────────┐
│                     视频播放流程图                          │
├─────────────────────────────────────────────────────────────┤
│                                                            │
│  启动程序                                                   │
│     │                                                      │
│     ▼                                                      │
│  ScanVideos() ──────► 扫描 video/ 目录                      │
│     │                                                      │
│     ▼                                                      │
│  ResetMPV() ────────► 初始化 MPV 实例                       │
│     │                                                      │
│     ▼                                                      │
│  PlayNextVideo() ───► 加载并播放第一个视频                  │
│     │                                                      │
│     ├──► TimerThread: 15分钟定时触发                        │
│     │                                                      │
│     └──► EventThread: 监听播放事件                         │
│              │                                             │
│              ├─ 播放完成 → PlayNextVideo()                 │
│              ├─ 错误发生 → ResetMPV() + PlayNextVideo()    │
│              └─ ESC 按键 → 退出程序                        │
│                                                            │
└─────────────────────────────────────────────────────────────┘
```

**关键代码逻辑**（VideoPlayer.cpp）：

```cpp
// 视频播放
bool PlayNextVideo() {
    EnterCriticalSection(&g_cs);
    
    // 获取下一个视频索引
    g_currentIndex = (g_currentIndex + 1) % g_videoFiles.size();
    
    // 构建播放命令
    std::vector<const char*> args;
    args.push_back("loadfile");
    args.push_back(WStringToUTF8(g_videoFiles[g_currentIndex]).c_str());
    
    // 发送到 MPV
    mpv_command(g_mpv, args.data());
    
    LeaveCriticalSection(&g_cs);
    return true;
}
```

### 3.3 多线程架构

**线程模型**：

| 线程 | 职责 | 同步机制 |
|------|------|----------|
| **主线程** | 消息循环、窗口管理 | - |
| **定时器线程** | 15分钟定时切换视频 | CRITICAL_SECTION |
| **事件线程** | MPV 事件监听与处理 | CRITICAL_SECTION + g_isResetting |

**线程安全设计**：

```cpp
// 全局状态保护
CRITICAL_SECTION g_cs;
bool g_isResetting = false;  // 重置标志（原子操作）

// 重置 MPV 时的同步
bool ResetMPV() {
    g_isResetting = true;  // 设置重置标志
    
    // 等待事件线程停止操作
    // ...
    
    // 销毁旧实例
    if (g_mpv) {
        mpv_terminate_destroy(g_mpv);
        g_mpv = NULL;
    }
    
    // 创建新实例
    g_mpv = mpv_create();
    // ...
    
    g_isResetting = false;  // 清除重置标志
    return true;
}
```

**避免竞态条件**：
- 使用 `CRITICAL_SECTION` 保护共享资源（播放列表、MPV 句柄）
- 使用 `g_isResetting` 标志协调线程间状态转换
- 延迟加载 DLL 避免初始化顺序问题

### 3.4 错误处理与恢复

**错误恢复机制**：

| 错误类型 | 检测方式 | 恢复策略 |
|----------|----------|----------|
| MPV 初始化失败 | `mpv_create()` 返回 NULL | 显示错误对话框，退出程序 |
| 视频文件不存在 | 播放列表为空 | 显示错误对话框，退出程序 |
| MPV 运行时错误 | `mpv_event` 事件回调 | 调用 `ResetMPV()` 重建实例 |
| 播放超时 | 定时器检测 | 自动切换到下一个视频 |
| 线程异常 | `g_mpvError` 标志 | 触发重置流程 |

**故障恢复流程**：

```
检测到错误
    │
    ▼
设置 g_mpvError = true
    │
    ▼
EventThread 检测到错误
    │
    ▼
调用 ResetMPV()
    │
    ▼
重建 MPV 实例
    │
    ▼
恢复播放（播放下一个视频）
```
